# S/4HANAについて

S/4HANAは2015年にリリースされたSAPの次世代ERPソリューションであり、従来のSAP ECCとはアーキテクチャが大きく異なります。

## 主な技術的変更点

- **Unicodeへの統一**  
  ECC時代はUnicodeシステムとNon-Unicodeシステムが混在していましたが、S/4HANAでは全てのシステムがUnicodeに統一されました。

- **Universal Journal（ACDOCA）への統合**  
  会計伝票がUniversal Journalテーブル（ACDOCA）に統合され、FI（財務会計）、CO（管理会計）、AA（資産会計）、ML（資材台帳）などのデータが1つのテーブルでリアルタイムに管理可能となりました。

- **Business Partner（BP）への統合**  
  得意先マスタと仕入れ先マスタがBusiness Partnerマスタに統合され、一元管理が実現しています。

## 移行推奨期限の変更

当初SAPは2025年までの移行を強く推奨していましたが、顧客の準備状況を考慮し、**2027年末**をメインサポート終了期限として設定しました。  
2026年現在、S/4HANA移行プロジェクトは最盛期を迎えており、今後数年でピークを迎えた後に徐々に落ち着くと予想されます。

## 今後の価値

技術的な知見や移行アプローチは、プロジェクト終了後も長く役立つスキルです。また、モジュールを横断した大規模プロジェクトの特性上、SAP全般に対する深い理解を得られる機会となります。
